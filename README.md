# speak-then-spell 🗣️→🔤

> Toddlers learn to speak years before anyone shows them the alphabet.
> We're doing the same with Python: **play with real, living code first — learn the grammar later.**

This is a two-player coding playground. No courses, no lectures, no "chapter 3: data types".
Every week there's a **mission** built from real open-source code. We both solve it
**separately**, push our solutions, and the site shows both attempts side by side.

## How the game works

1. **A mission drops** in `playgrounds/week-XX-<name>/MISSION.md`.
   It always contains a piece of *working* code lifted or adapted from a real,
   popular Python project — plus a challenge to break it, fix it, or upgrade it.
2. **We solve it alone.** No peeking. Your solution goes in
   `playgrounds/week-XX-<name>/solutions/<your-name>/`.
3. **Push whenever you're done.** First one to push gets bragging rights, not help requests.
4. **Rebuild the site** and compare:
   ```bash
   python build_site.py
   git add . && git commit -m "week XX solved" && git push
   ```
5. **Talk about it.** The differences between two solutions to the same problem
   teach more than any tutorial. That's the whole point.

## Rules of the playground

- **Run first, read docs later.** If you don't know what a line does, run it, break it, print it.
- **No solution is wrong if it runs.** Ugly-but-working beats elegant-but-imaginary.
- **Steal from the pros.** Missions come from real repos (requests, rich, flask, tqdm...).
  Reading production code is the "listening to adults talk" phase of toddler learning.
- **One week, one toy.** Each mission focuses on one idea (walrus `:=`, exceptions,
  generators, decorators...) hiding inside real code.

## Seeing the results (GitHub Pages)

The `docs/` folder is a tiny static site generated by `build_site.py`.
To turn it on, once, in the GitHub repo settings:

**Settings → Pages → Source: "Deploy from a branch" → Branch: `main`, folder: `/docs` → Save.**

After that, every push updates the scoreboard at `https://<username>.github.io/speak-then-spell/`.

## Setup for player 2 (that's you, Mission)

```bash
git clone <this-repo-url>
cd speak-then-spell
# your folder is already waiting: playgrounds/week-01-the-walrus-hunt/solutions/mission/
```

Yes — every challenge file is called `MISSION.md`. Named after you. Obviously. 😌
